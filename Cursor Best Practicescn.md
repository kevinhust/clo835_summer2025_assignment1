# Cursor 最佳工作方法

## 1. 规划与协作
- 在使用 Cursor 之前，让 Claude 用 Markdown 创建一个清晰详细的计划（让它提出澄清性问题，然后批评自己的计划，再重新生成）。将其添加到 Instructions.md 文件中（这样你可以经常让 Cursor 参考它）。
- 让 ChatGPT 规划，Cursor 编码：告诉 ChatGPT 你想要创建什么，让它为另一个负责编码的 AI 提供指令，然后将所有内容粘贴到 Cursor Composer Agent 中。ChatGPT 增加了另一层规划，从而降低了遇到问题的几率。
- 遇到问题时，让 Cursor 写一份报告，列出所有文件及其功能，并描述遇到的问题，发给 Claude 或 ChatGPT 要解决方案。

## 2. 规则与增量开发
- 使用 `.cursorrules`（它们总是在 AI 上下文中）来定义大致的规则。参考：https://cursor.directory
- 增量开发流程：
  1. 定义一个小任务进行增量。
  2. 编写（或让 AI 编写）一个在此增量中失败的测试用例。
  3. 指示 AI（通常在 agent 模式下）编写代码以通过测试。
  4. 如果测试失败，AI 分析故障原因并尝试修复代码，循环回到步骤 3。
  5. 一旦测试通过，开发者审查更改。
- 示例：先编写测试，然后编写代码，再运行测试并更新代码，直到测试通过。让 agent 以小段“编辑-测试”循环的方式逐步编写代码。

## 3. 提示与上下文管理
- 鼓励链式思维，在 prompt 中明确表达。
- 保持上下文简短：通过 @ 明确添加文件来保持上下文简短。上下文越长，AI 提供的细节就越多。当上下文变长时，开始新聊天。
- 频繁地重新同步 / index 代码。
- 使用 `.cursorsignore` 来排除不相关的文件。
- 使用 / Reference 打开的编辑器快速将它们添加到上下文中。

## 4. 工具与资源
- 使用 gitinsect.com，将所有脚本配置和相关文件（可按扩展名筛选）整合到一个页面中。
- https://context7.com/ 用于参考最新文档。
- 经常使用 git 进行版本控制，避免出现太多未提交的更改。

## 5. 记事本与命令
- 记事本是常用的提示符。
- 示例：启用 Yolo 模式以便它编写测试。允许任何测试，例如 vitest、npm test、nr test 等，也可以使用基本构建命令，例如 build、tsc 等。也始终允许创建文件和目录，如 touch、mkdir 等。

## 6. 系统提示与表达
- 可选：在光标设置中的“AI 规则”中设置系统提示。
- 保持 prompt 简洁明了：
  - 使用替代词汇
  - 避免不必要的解释
  - 优先考虑技术细节而不是通用建议
